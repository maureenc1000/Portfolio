This project presents a hypothetical software system proposal supported by a fully designed relational database. The case study demonstrates the process of translating business requirements into a structured database solution using Entity-Relationship (ER) modeling and SQL implementation.

The goal of the project is to illustrate best practices in database design, normalization, and SQL-based database management, including creation, maintenance, and querying of relational data.

The primary objectives of this project are to:
- Design a relational database schema based on a proposed software application
- Create an Entity-Relationship (ER) diagram representing system entities and relationships
- Implement the database using SQL Data Definition Language (DDL)
- Demonstrate data maintenance operations using SQL
